# 概念的な機能要件・非機能要件
- 主幹の提供価値はローカル完結で、情報漏洩リスクが低いこと。

## 文字起こしのチャンクのつなぎ目について

５秒ほど音声ファイルを物理的に重ねることで、切れ目をなくした。また、そうすれば文字起こしがちょっとおかしくなるけど、文字お越しにそんなものがあっても誰も気にしないし、文字起こしファイルはバックアップで人が見るようなものは議事録生成機能が構築した議事録なので、そのおかしさは許容すべき。

## AIに関して
- 混合精度で推論を行うことで、VRAM使用量を削減しつつ、推論速度を向上させる。また、コストパフォーマンスが良いので

- 使用するAIモデルの選択肢はライブラリの使用可能モデル一覧を取得しておいて、それをそのまま選択肢にする形にしたいです。

## 必要な機能
- 文字起こしテキストのRAG
- RAGチャットボット
- ファイルからの文字起こし機能
- リアルタイムでの文字起こし機能
- LLMを使った文字起こしデータの清書や要約、議事録に変換
- 自動的にマシンスペックを検知した最適なローカルAIモデルを使って動作させる。
- fastEmbeddを使って、ローカルで完結するEmbedding能力
- OllamaとfastEmbedを使用すればデータが一切外に漏れないアーキテクチャ
- LLMによる自動タグ生成機能とそのタグをメタデータに付与してデータベースに保存する機能
- AIチャットボット機能には、どのタイトルの情報を参考にしたのかっていうのを出力するように改良するのと、また、この改良はプロンプトエンジニアリングではなくシステムで自動挿入するようにすること。

### メモ
この選択肢に、その他っていうのがデフォルトで存在するようにしてください。また、「新規プロジェクト」っていう選択肢も構築して、その場合横のテキストボックスにプロジェクト名を入力する形にお願いします。また、横の新規プロジェクト名入力のテキストボックスは、新規プロジェクトって選択しないと出現しないようにしてください。

ファイルからの録音機能と、リアルタイムの機能でも、プロジェクトの機能を使えるようにしてください。

履歴UIにプロジェクト絞り込み機能と絞り込みを解除する機能を実装します。

設定UIからプロジェクトの削除を行えるようにしたいです。プロジェクトを削除するとそのプロジェクトに属していたデータは自動的にその他のプロジェクトが付与されるようにしたいです。

物理カット ＋ LLM接合: 無音検知よりも安定性が高く、ローカル環境では最適。（現状維持）
ハードウェアID暗号化: 物理的盗難には強いが、オープンソース環境では気休めになりがち。
リカバリーの課題: ローカル暗号化には「パスワード忘れ」によるデータ紛失リスクが付き物。
