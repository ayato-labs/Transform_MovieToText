# Transform MovieToText 設計思想・開発スタンス

**超重要:巨人の肩の上に乗れ**
MITライセンスなどの緩いライセンスのオープンソースや、無料で提供してくれている既存機能やAPIなどを活用して、車輪の再開発を避けること。そのためにリサーチをよくすること

## 1. 開発の基本スタンス (As-Is Philosophy)
本作は**「徹頭徹尾、自分自身が便利に使うためのツール」**として生み出されたものです。
自分が欲しい機能を作り、自分が快適に動かせる最高のアーキテクチャを追求しています。

その恩恵をコミュニティと共有するためオープンソースとして公開していますが、本質的には「ついでに公開してやるぜ」というスタンスをとっています。

### 1.1 手段としての自動化提供
「他の人の環境構築ができなくて使えない」という事態は、エンジニアの美学として許容できません。そのため、`run.bat` と `uv` を用いた **Thin Client(スマートインストーラー)方式** のような、「誰でもワンクリックで環境が整う」強力な仕組みは積極的に提供します。

### 1.2 サポートの境界線
しかし、以下の点については明確に割り切ります。
- **個別の環境依存バグへの対応**: ユーザー固有のOSの癖、ネットワークの制限、ハードウェアの相性などでバグが発生しても、「おれには全く関係ない」という自己責任の原則を貫きます。
- **手厚いユーザーサポートの不在**: 無料で公開されているツールに対し、過度なサポートや手取り足取りの対応を求めるのはご遠慮ください。

## 2. マネタイズとサポート体制の変化
現在提供している高度な自動構築スクリプトや機能は善意と技術的興味によるものですが、このスタンスが変化する唯一の例外があります。
それは**「本システムをマネタイズ(収益化)のフェーズへ移行させた時」**です。

有償版、または商用サポートを提供する段階になった場合には、当然のことながらプロとしてユーザーの環境バグや要望にコミットする責任を負います。
逆を言えば、無料公開の現状においては「As-Is(現状有姿)」であり、その他のケースについては関知しない(知らん)というスタンスです。

## 3. なぜこのスタンスなのか?(AIデスクトップアプリの現実)
数GBの巨大なライブラリ(PyTorch、CUDA等)を抱え、ローカルのGPUや複雑なPython環境に依存するモダンなAIデスクトップアプリを配布する際、すべてのユーザー環境を完璧に保証することは事実上不可能です。

開発者が無限のサポート対応に疲弊し、開発自体が止まってしまうことはOSSにとって最大の損失です。
「俺の環境で最速・最強で動く。使いたい奴は使え」
この尖ったスタンスこそが、個人開発のAIツールを健全かつサステナブルに進化させ続けるための最適解だと考えています。
